
# LatamCreativa - Project Handover

## Overview
LatamCreativa is a creative platform designed for artists and developers in Latin America. It features a dual-mode interface ("Creative" and "Dev") that adapts the content and theme based on the user's focus.

## Tech Stack
- **Framework**: React 18+ (Router v6.4+)
- **Styling**: Tailwind CSS (Dark Mode enabled)
- **Icons**: Lucide React
- **Bundler/Runtime**: Vite (implied by file structure)

## Key Features & Implementations

### 1. Dual Mode System
The app switches between `creative` and `dev` modes via `useAppStore`. This toggles:
- **Theme Colors**: Amber/Orange for Creative, Blue/Indigo for Dev.
- **Content Filtering**: Feeds and lists filter items based on the `domain` property ('creative' vs 'dev').
- **Navigation**: Secondary sidebar links change based on mode.

### 2. Navigation & Routing
- Uses `react-router-dom` v6 data router.
- `MainLayout` wraps the app with the Header, Sidebars, and Global Overlays (Chat, Modals).
- Detail views use dynamic parameters (e.g., `/portfolio/:id`) and hook into `useParams` to fetch data.

### 3. Data Management
- Currently uses static mock data located in `src/data/`.
- Data is split by domain (jobs, events, etc.) and aggregated in `content.ts`.
- **Note**: In a real implementation, `useAppStore` or React Query would replace direct imports to fetch from an API.

### 4. Global State (`useAppStore`)
- Manages UI state: Sidebar visibility, Content Mode, Cart Items, Search Query, and User Notifications.
- Actions are exposed to modify state from any component.

## Known Issues / To-Dos
1. **Mock Data**: All data is static. Needs API integration.
2. **Auth**: User profile is mocked. No real authentication flow exists.
3. **Persistance**: Cart and settings reset on reload. Needs `localStorage` or backend persistence.
4. **Performance**: Large lists (Feed) are not virtualized.

## How to Extend
1. **Add a new Module**:
   - Create `src/views/NewModuleView.tsx`.
   - Add route to `src/router.tsx`.
   - Update `src/data/navigation.ts` to add sidebar link.
2. **Add Data**:
   - Create new interface in `types.ts`.
   - Add mock data file in `src/data/`.
   - Export via `src/data/content.ts`.
